# Using Tegel Lite (CSS)

Tegel Lite is a pure CSS alternative to Tegel's component library, providing the same design system without relying on Web Components or Stencil. It's a styles-only component system built using SCSS and compiled to CSS files, aligned to the same design tokens and variables as the original Tegel.

## When to Use Tegel Lite

Use Tegel Lite when you:
- Want to avoid JavaScript frameworks
- Need a lightweight, CSS-only solution
- Are building server-rendered applications
- Prefer full control over HTML structure and behavior

Tegel Lite works with any framework or vanilla JavaScript. However, for React or Angular applications, we recommend using the framework-specific packages (`@scania/tegel-react` or `@scania/tegel-angular`) which provide a better developer experience with web component wrappers.


## Installing `@scania/tegel-lite`

Install the `@scania/tegel-lite` package via npm.

```bash
npm install @scania/tegel-lite
```

## Theming

Tegel Lite supports both **Scania** and **Traton** themes. Set the theme by adding a class to the `<html>` element (or any parent element):

```html
<!-- Scania (default) -->
<html class="scania">

<!-- Traton -->
<html class="traton">
```

## Setup

Import the global styles in your CSS file:

```css
@import '@scania/tegel-lite/global.css';
```

Import only the component styles you need:

```css
@import '@scania/tegel-lite/tl-button.css';
@import '@scania/tegel-lite/tl-header.css';
```

## Usage

Tegel Lite components use BEM naming conventions with `tl-` prefix:

```html
<button class="tl-button tl-button--primary tl-button--md">
  <span class="tl-button__label">Click me!</span>
</button>
```

## HTML Example

For vanilla HTML projects, link to the CSS files:

```html
<!DOCTYPE html>
<html lang="en" class="scania">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Tegel Lite</title>
  <link rel="stylesheet" href="./node_modules/@scania/tegel-lite/global.css">
  <link rel="stylesheet" href="./node_modules/@scania/tegel-lite/tl-button.css">
</head>
<body>
  <button class="tl-button tl-button--primary tl-button--md">
    <span class="tl-button__label">Click me!</span>
  </button>
</body>
</html>
```

## BEM Naming

Tegel Lite uses BEM (Block Element Modifier) naming conventions:

- **Block**: `.tl-button`
- **Element**: `.tl-button__label`
- **Modifier**: `.tl-button--primary`

### Important: Correct Class Usage

BEM modifiers like `.tl-button--primary` are **not standalone** â€” they must always be used together with the base class.

**Correct:**
```html
<button class="tl-button tl-button--primary tl-button--md">
  <span class="tl-button__label">Click me!</span>
</button>
```

**Incorrect:**
```html
<!-- Missing base class -->
<button class="tl-button--primary">
  <span class="tl-button__label">Click me!</span>
</button>
```

Always include the base class (`.tl-button`) when using modifiers like `--primary`, `--disabled`, `--lg`, etc. Modifiers extend the base styling, they don't replace it.

## Component Documentation

Detailed documentation for all components is available in the **Tegel Light (CSS)** section in Storybook, including:

- Usage examples with HTML structure
- Available elements and modifiers
- Required stylesheets
- JavaScript integration examples (when needed)

## JavaScript Requirements

Some components (Accordion, Dropdown, Modal, Tabs, Tooltip) require JavaScript for interactive behavior such as:
- Toggling visibility states
- Handling user interactions (clicks, keyboard events)
- Managing ARIA attributes for accessibility
- Position calculations

These components provide the base styling and structure, but you need to implement the interactive behavior yourself. Check the component documentation in the **Tegel Light (CSS)** section for detailed requirements and examples.

## Local Development Setup

If you want to use a local version of Tegel Lite during development, you can link the package:

### Using npm link

In the `tegel/packages/tegel-lite` directory:

```bash
npm link
```

In your project:

```bash
npm link @scania/tegel-lite
```
