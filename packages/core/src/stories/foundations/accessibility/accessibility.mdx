## 🛠️ What Was Done in A11y for Tegel

<div style={{
  border: '1px solid #d1ecf1',
  borderRadius: '8px',
  padding: '20px',
  marginBottom: '20px',
  backgroundColor: '#d1ecf1'
}}>

### Core Accessibility Features
- **ARIA Attributes Support**: All components support standard ARIA attributes.
- **Keyboard Navigation**: Full keyboard support for all interactive components.
- **Screen Reader Compatibility**: Components are designed to enable seamless interaction with screen readers.
- **Focus Management**: Proper focus handling and visual focus indicators.
- **Color Contrast**: All components meet WCAG 2.1 AA color contrast requirements (except for the `XXX XXX` component).
- **Semantic HTML**: Components use appropriate semantic HTML elements.

### Automated Testing Infrastructure
- **axe-core Integration**: All components are tested using axe-core accessibility testing library
- **Playwright Integration**: Automated accessibility tests run on every build
- **CI/CD Pipeline**: Accessibility tests are part of the continuous integration process
- **Storybook A11y Addon**: Interactive accessibility testing in development environment

## 📋 Accessibility Guidelines

<div style={{
  border: '1px solid #e1e5e9',
  borderRadius: '8px',
  padding: '20px',
  marginBottom: '20px',
  backgroundColor: '#ffffff'
}}>

Tegel components are designed to meet and exceed the following accessibility standards:

### Reporting Issues
If you encounter accessibility issues with any Tegel component:

1. **Create an Issue**: Report problems in the [GitHub repository](https://github.com/scania-digital-design-system/tegel/issues)
2. **Include Details**: Provide specific information about the accessibility barrier
3. **Test Environment**: Mention browser, assistive technology, and component version
4. **Expected Behavior**: Describe what should happen for proper accessibility

### Best Practices for Implementation
When using Tegel components:

1. **Provide Labels**: Always include meaningful and descriptive labels for form controls
2. **Use ARIA Attributes**: Leverage built-in ARIA support for components that support it.
3. **Test Regularly**: Use both automated and manual testing
4. **Consider Context**: Ensure components work within your application's accessibility strategy.

</div>
